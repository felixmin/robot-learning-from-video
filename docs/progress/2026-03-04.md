- several official `lerobot/...` OXE ports had `meta/episodes` video timestamps extending past the actual mp4 duration
- this is a dataset-port issue, not a sampler or decoder bug

- `lerobot/taco_play`

## Follow-up: mixed-camera fail-fast and explicit non-changes

Applied one additional fail-fast constraint in `LeRobotV3DataModule.setup()`:

- every configured image role in `data.request.image_requests` must be mapped by every source
- each mapped key must exist in source metadata features and must be image/video typed

Rationale:

- current collation requires identical image-key sets across all samples in a batch
- allowing optional image roles to be absent per source leads to runtime batch inconsistencies
- failing in setup is clearer and simpler than runtime partial handling

Intentionally unchanged (documented tradeoff):

- validation still uses weighted random sampling (not an exhaustive val pass)
- run normalization stats are still built from source metadata stats (not train-split recomputation)
